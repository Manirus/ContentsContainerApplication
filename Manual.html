Manual Testing


1. what is Software Testing?
  • Software Testing is a method to check whether the actual software product matches expected requirements and to ensure that software product is defect free.
  • PurposeIdentify Errors, gaps or missing requirements in contrast to actual requirements.
  • Benefits:

    2. 7 Principles of Software Testing?
  • Testing shows the presence of defects, not their absence
  • Exhaustive testing is not possible
  • Early testing
  • Defect clustering
  • Pesticide paradox
  • Testing is context dependent
  • Absence of errors fallacy
    • 3. Software Build A software build is the process of converting source code files into an executable file that can be run on a computer.
    4. Retesting Retesting is the act of testing again a previously tested product or system to verify that it still meets required specifications and standards.
    5. Types of Testing
  • Static Testing
  • Software Application is tested without code execution. Main Objective-Improve the Quality of Software Applications by finding errors in early stages of Software Development process.
  • Dynamic Testing
  • Code is executed. It checks for functional behavior of Software System, Memory/CPU usage and overall performance of the system. Main Objective-Software Product works in conformance with the business requirements.
    		•
    6. Unit Testing Type of Software Testing where individual units or components of a software are tested. Purpose-Validate that each unit of the software code performs as expected. Unit Testing done by the developers.
    7. Types of System Testing
  • Functional Testing
  • Non-Functional Testing or Performance Testing
    • 8. Functional Testing Functional testing is a type of software testing that focuses on verifying that a software product meets its functional requirements and specifications.
    9. Non-Functional Testing Non-functional testing is a type of testing that evaluates a system's non-functional requirements, such as performance, reliability, scalability, security, etc. It focuses on how well the system performs under certain conditions, rather than on the functional requirements. Non-functional testing helps to ensure that the system is fit for its intended use and can handle the expected load and usage.
